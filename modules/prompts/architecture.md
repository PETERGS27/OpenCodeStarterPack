# Architecture Prompt

## System design workflow

### 1. Requirements
- Functional: what must the system do?
- Non-functional: performance, scalability, reliability, security
- Constraints: budget, time, team size, tech stack

### 2. Scope
- What's in scope for this design?
- What's explicitly out of scope?
- Success criteria

### 3. High-level design
- System components and their responsibilities
- Data flow between components
- API boundaries
- Storage: what data, where, and how

### 4. Deep dive
- For each component: internal architecture
- Data models and schemas
- Key algorithms and data structures
- Error handling and edge cases

### 5. Trade-offs
- What alternatives were considered?
- Why was this approach chosen?
- What are the downsides of this design?

### 6. Evaluation
- Does it meet the requirements?
- What's the bottleneck?
- How does it scale?
- How does it fail?
